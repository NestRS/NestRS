You spend the next nine pages shipping one feature — a `users` resource —
from an empty workspace to a running binary that authenticates callers,
filters reads to their organisation, validates inputs, persists through
a real Postgres, exposes the same data over GraphQL, and ships an e2e
test that boots the whole module against a throwaway database. Each
page leaves you with code that compiles and a command you can run.

## What you end up with

A workspace shaped like the reference one — `apps/tutorial-api/` for the
binary and a `users/` feature with one port and two transports.

<FileTree>
- apps/tutorial-api/
  - Cargo.toml
  - src/
    - main.rs
    - module.rs
  - tests/
    - e2e.rs
- crates/
  - features/
    - src/
      - users/
        - entity.rs
        - service.rs
        - module.rs
        - mod.rs
        - http/
          - controller.rs
          - module.rs
          - mod.rs
        - graphql/
          - resolver.rs
          - module.rs
          - mod.rs
</FileTree>

Two `curl`s and one GraphQL query against the running binary:

```bash frame="terminal"
$ curl -s -H 'Authorization: Bearer <token>' http://localhost:3000/users/<id>
{"id":"018f…","name":"Ada","email":"ada@acme.test"}

$ curl -s -H 'Authorization: Bearer <token>' http://localhost:3000/graphql \
    -H 'Content-Type: application/json' \
    -d '{"query":"{ users { id name org { id } } }"}'
{"data":{"users":[{"id":"018f…","name":"Ada","org":{"id":"018e…"}}]}}
```

And a passing e2e suite:

```bash frame="terminal"
$ just test-e2e
running 1 test
test users_round_trip_under_a_bearer_token ... ok
```

## What you should know going in

The tutorial assumes you've already followed
[Getting started](/getting-started/) and skimmed
[Fundamentals](/fundamentals/), [Configuration](/configuration/), and
[HTTP](/http/) — the toolchain is installed, `just dev` runs the bare
baseline, and you know what a module, a provider, and a controller are.
`docker` is available for the Postgres step.

Rust 1.75+ is enough. Pages 7 and 9 (database, testing) need a Postgres
the workspace already provisions with `just db up`; pages 2 to 6 run
without one.
